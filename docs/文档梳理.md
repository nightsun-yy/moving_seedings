# Moving Seedlings 文档梳理

更新日期：2026-04-14

这份文档的目标不是把所有依赖文件逐个展开，而是把当前仓库里“真正需要理解和维护的文件”梳理清楚，并把“自动生成目录 / 缓存目录 / 外部依赖目录”单独标出来，方便后续继续整理。

## 1. 先看结论

- 这个仓库当前主要由 4 条线组成：`前端展示`、`PLC 监控`、`PPT 生成`、`需求与模型文档`。
- 真正长期需要关注的核心文件并不多，主要是：
  - `plc_monitor.py`
  - `docs/基本需求.md`
  - `frontend/src/App.tsx`
  - `tools/create_ppt.py`
  - `docs/PPT/build_speech_ppt.py`
- 看起来“很乱”的主要原因，不是源码太多，而是仓库里混放了以下几类内容：
  - 运行依赖：`frontend/node_modules/`、`.pptx_deps/`
  - 构建产物：`frontend/dist/`
  - Python 缓存：`__pycache__/`、`tools/__pycache__/`、`docs/PPT/__pycache__/`
  - 图片缓存：`assets/ppt/.cache/`、`docs/PPT/.cache/`
  - 下载中转目录：`downloads/`
- 当前 3D 模型文件有“原始来源”和“前端运行副本”两套：
  - 原始来源在 `docs/*.gltf`
  - 前端实际运行时读取的是 `frontend/public/models/master-layout.gltf`

## 2. 顶层目录总览

| 路径 | 作用 | 是否建议手动维护 |
| --- | --- | --- |
| `.git/` | Git 仓库元数据 | 否 |
| `.pptx_deps/` | 本地放置的 `python-pptx` 依赖包，供 Python 脚本直接引用 | 否，除非升级依赖 |

## 8. 当前“乱”的主要来源

如果从“是否需要你手动维护”的角度看，当前仓库最容易造成混乱的是以下目录：

| 路径 | 混乱原因 | 建议 |
| --- | --- | --- |
| `frontend/node_modules/` | 文件量巨大 | 保留即可，不要手动进入逐文件看 |
| `frontend/dist/` | 打包后自动生成 | 只在部署或验证时关注 |
| `.pptx_deps/` | 第三方库 vendor 目录 | 视为依赖，不要当业务代码读 |
| `downloads/` | 下载包、测试输出、中转工具混在一起 | 可以后续改成 `downloads/archive/` 或单独清理 |
| `assets/ppt/.cache/` | 裁图缓存 | 可以删，脚本会重建 |
| `docs/PPT/.cache/` | PPT 生成缓存 | 可以删，脚本会重建 |
| `__pycache__/`、`tools/__pycache__/`、`docs/PPT/__pycache__/` | Python 自动生成 | 可以忽略或清理 |

## 11. 一句话版本

这个仓库现在其实不是“代码很多”，而是“源码、文档、模型、生成文件、依赖文件都混放在一个层级里”。真正的主线很清楚：

- 业务说明在 `docs/`
- 前端页面在 `frontend/`
- PLC 监控在 `plc_monitor.py`
- PPT 生成在 `tools/` 和 `docs/PPT/`
- 大量看起来杂乱的内容，其实是依赖、缓存和导出产物

如果后续继续整理，建议优先把“模板残留文件”和“下载 / 缓存目录”再收一轮。
